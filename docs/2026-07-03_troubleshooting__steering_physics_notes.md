### 1. 조향 각도 단위 — `compiler angle`

#### 증상
`ctrl=0.5`(목표 28°)를 줘도 앞바퀴가 약 0.5°만 꺾이고, 액추에이터 힘(`actuator_force`)만
최대치(약 23.6 N·m)로 쌓인 채 조인트가 움직이지 않았다.

#### 원인
XML에 `<compiler>` 태그가 없으면 MuJoCo는 기본값 **`angle="degree"`** 를 적용한다.
즉 XML 안의 모든 각도 숫자를 도(degree)로 해석한다.

- `euler="90 0 0"` → 90도 (바퀴 축 정렬, 의도대로 정상)
- `joint range="-0.5 0.5"` → **±0.5도로 오해석** (라디안 ±0.5 = ±28°를 의도했음)

즉 라디안 의도값(`range`)과 도 의도값(`euler`)이 한 파일에 섞인 게 근본 문제였다.
조인트가 ±0.5도에 갇히자, 막힌 조향력의 반작용 토크가 chassis에 yaw로 전달돼
"좌우만 눌러도 차체가 도는" 증상까지 나타났다.

#### 진단
```python
m = mujoco.MjModel.from_xml_path('envs/assets/rover.xml')
jid = mujoco.mj_name2id(m, mujoco.mjtObj.mjOBJ_JOINT, 'steer_fl_joint')
print(m.jnt_range[jid])   # 내부 저장은 항상 rad. ±0.0087이면 degree로 해석된 것
```

#### 해결
`<compiler angle="radian"/>` 을 명시해 모든 각도를 라디안으로 통일했다. 대신 `euler`는
라디안으로 바꿔야 한다.

- `euler="90 0 0"` → `euler="1.5708 0 0"` (π/2)
- `range="-0.5 0.5"`, `ctrlrange="-0.5 0.5"` → 그대로 두면 진짜 ±0.5 rad (±28°)

#### 정리
**MuJoCo는 `<compiler>`가 없으면 각도를 degree로 읽는다.** 라디안 값을 쓰려면
`angle="radian"`을 반드시 명시한다. qpos/jnt_range 등 내부 저장값은 단위 설정과 무관하게 항상 라디안이므로, 진단할 때 내부값과 XML 표기값을 혼동하지 않는다.

### 2. 저중력에서 튀는 현상 — integrator & solref

#### 증상
로버가 정지 상태에서도 계속 통통 튀었다.

#### 원인
달 중력(`-1.62`)은 지구의 약 1/6이라, 접촉 보정으로 생긴 작은 반발 에너지가 훨씬 느리게 감쇠한다. 게다가 기본 적분기(Euler)는 조인트 감쇠·액추에이터가 있을 때 수치적으로 불안정해 에너지가 쌓이는 경향이 있다.

#### 해결
```xml
<option timestep="0.005" gravity="0 0 -1.62" integrator="implicitfast"/>
...
<geom friction="0.4 0.01 0.01" solref="0.02 1.5"/>   <!-- default geom -->
```

- `integrator="implicitfast"`: 감쇠·액추에이터가 있을 때 Euler보다 안정적
- `solref="0.02 1.5"`: 접촉을 살짝 과감쇠(dampratio > 1)시켜 반발을 억제

### 3. 조향감 — `kp`

#### 증상
조향이 딱딱하게 확 꺾이고, 좌우 방향이 부드럽게 전환되지 않았다.

#### 원인
`position` 액추에이터는 `force = kp × (목표각 − 현재각)` 으로 동작한다. 기존 `kp=50`은
너무 세서 목표각에 거의 즉시 도달했다.

#### 해결
`kp=50 → 15`로 낮췄다. `kp`가 클수록 목표각에 즉시 도달해 딱딱하고, 낮출수록 서서히
수렴해 부드럽다. `damping=2.0`과 짝을 이뤄 진동 없이 수렴하며, 조향각이 지면 저항에
밀려 덜 꺾이면 `kp`를 높이고 여전히 딱딱하면 낮춘다.

`kp`는 물리 모델(XML)의 일부라, 이 값을 조정하면 **manual_drive와 강화학습이 동일한
조향 다이나믹스를 공유**한다. (부드러움을 manual_drive 코드의 rate-limit으로 넣으면
RL은 여전히 딱딱한 조향을 써서 감각이 어긋난다. 그래서 코드가 아닌 `kp`로 해결했다.)

### 4. 관성 주행 — velocity 액추에이터의 제동

#### 증상
전후진 키를 떼면 몸체가 앞으로 가는 와중에도 바퀴는 즉시 정지한다. 의도적인 토크가 없다면 관성으로 굴러가야 한다.

#### 원인
`velocity` 액추에이터는 `ctrl=0`을 주면 "속도 0을 유지"하려고 능동적으로 제동 토크를 건다. 즉 `ctrl=0`은 브레이크지 중립이 아니다.

#### 해결
키를 떼면 `ctrl`을 0이 아니라 **현재 바퀴 속도**(jointvel 센서값)로 설정한다. 그러면 오차(목표−현재)가 0이 되어 토크가 0이 되고, 바퀴가 관성으로 자연 감속한다.

```python
# manual_drive.py — 키를 놓았을 때
data.ctrl[2] = data.sensor('vel_fl').data[0]   # 나머지 바퀴도 동일
```
